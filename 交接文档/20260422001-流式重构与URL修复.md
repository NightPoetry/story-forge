## 关键设计决策与原因

### 为什么不修 Function Call 流式，而是降级到纯文本模式？

Function Call arguments 是否增量流式由**服务端 API 实现决定**，客户端无法控制。glm-5、部分 DeepSeek 代理等都会一次性发完 arguments。纯文本 XML 模式（`<write_story>内容</write_story>`）走的是普通 `content` 字段，所有 API 都能正常逐 token 流式。

### 为什么 textarea 在流式期间改为 uncontrolled？

React 的 controlled textarea（`value={...}`）会在每次 re-render 时把 DOM value 重置为 store 中的值。流式期间直接写 `el.value = text` 绕过 React 获得零延迟反馈，但下一次 React re-render 就会覆盖。移除 `value` prop 后 React 不再控制 textarea 内容，流式结束后通过 `prevStreamingRef` 检测 isStreaming 转 false 来同步。

### 滚动位置保持的实现方式

`el.style.height = 'auto'` 会使 textarea 高度瞬间归零，导致父容器 scrollHeight 突变，浏览器重置滚动位置。解法：设高度前读 `parentElement.scrollTop`，设完后写回。

## 调试 API 使用手册（后续维护者参考）

启动 `npm run dev` 后可用：

```bash
# 检查连接状态
curl http://localhost:1420/__debug/ping

# 读取前端日志（可按 tag/level/since 过滤）
curl "http://localhost:1420/__debug/logs?tag=postSSE&limit=20"

# 在浏览器中执行 JS
curl -X POST http://localhost:1420/__debug/eval \
  -H 'Content-Type: application/json' \
  -d '{"code": "return window.__STORE__.getState().apiModel"}'

# 指定只在 Tauri 窗口中执行
curl -X POST http://localhost:1420/__debug/eval \
  -H 'Content-Type: application/json' \
  -d '{"code": "return navigator.userAgent", "target": "tauri"}'

# 清空日志
curl http://localhost:1420/__debug/logs/clear
```

Mock SSE 端点用于测试流式链路而不需要真实 API key：
- `POST /__debug/mock-sse/chat/completions` — 正常 50ms 间隔流式
- `POST /__debug/mock-buffered/chat/completions` — 一次性发完（模拟 WKWebView 缓冲）
