```bash
npm run tauri:build
# 产物在 src-tauri/target/release/bundle/
```

## 当前三平台配置（.github/workflows/release.yml）

| 平台 | Runner | 产物 | 特殊参数 |
|------|--------|------|---------|
| macOS | macos-latest | .dmg（ARM+Intel 通用） | `--target universal-apple-darwin` |
| Linux | ubuntu-22.04 | .AppImage / .deb | 无 |
| Windows | windows-2022 | .exe（NSIS） | `--bundles nsis` |

> Windows 用 `windows-2022` 不用 `windows-latest`，后者 WiX 对中文路径有 bug。  
> Windows 用 NSIS 不用 MSI，WiX light.exe 处理中文 productName 路径会崩。

## 已踩过的坑（勿重蹈）

### 1. fs 权限：Tauri 2 权限分两层，缺一不可

Tauri 2 的 fs 权限分两层，打包后严格执行：
- **命令权限**（`fs:allow-write-text-file` 等）：允许调用该命令
- **路径 scope**：允许访问哪些路径

两者都要有，操作才能执行。开发模式对 scope 更宽松，所以只在打包后暴露。

**当前 capabilities 配置逻辑：**

| 操作 | 路径 | 覆盖权限 |
|------|------|---------|
| 项目新建/读写/删除 | `BaseDirectory.AppData` | `fs:allow-app-read-recursive` + `fs:allow-app-write-recursive` |
| 导出故事/备份 | dialog 选择的任意路径 | `fs:allow-write-text-file` + scope `$HOME/**` |
| 导入备份 | dialog 选择的任意路径 | `fs:allow-read-text-file` + scope `$HOME/**` |

**注意：** 新增 fs 功能时，除了加命令权限，还要确认路径 scope 是否覆盖。`$HOME/**` 覆盖桌面/文档/下载等常见位置。

### 2. 图标必须在 tauri.conf.json 显式列出
图标文件放在 `icons/` 目录还不够，必须在 `bundle.icon` 数组里写明，否则报 `Couldn't find a .ico icon`。  
**当前已配置：** 32x32.png / 128x128.png / 128x128@2x.png / icon.png / icon.icns / icon.ico

### 3. tauriScript 不能包含 build 子命令
`tauri-action` 会自动在 `tauriScript` 后追加 `build`，所以只能填 CLI 入口：
```yaml
tauriScript: npx @tauri-apps/cli   ✓
tauriScript: npm run tauri:build   ✗  （会变成 tauri build build）
```

### 4. workflow 必须声明写权限
```yaml
permissions:
  contents: write   # 缺少这行会导致 Release 创建失败
```

### 5. SSH 22 端口被封时
```bash
ssh-keyscan -p 443 ssh.github.com >> ~/.ssh/known_hosts
# ~/.ssh/config 加：
Host github.com
  Hostname ssh.github.com
  Port 443
  User git
```

## 关键文件位置

| 文件 | 作用 |
|------|------|
| `src-tauri/tauri.conf.json` | 应用名、版本、图标列表、窗口配置 |
| `src-tauri/capabilities/default.json` | 权限配置（fs/dialog/shell） |
| `.github/workflows/release.yml` | 三平台自动构建 workflow |
| `src-tauri/icons/` | 所有图标文件 |

## 版本号

`tauri.conf.json` 里的 `version` 字段和 tag 名保持一致，打包前记得更新。
